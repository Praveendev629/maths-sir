# Setup Guide - Generate MindAR Training Data

## Step-by-Step Guide to Generate `.mind` Files

### Option 1: Using MindAR Compiler (Recommended)

#### Install Node.js Package Globally
```bash
npm install -g @mnstc/mindar-compiler
```

#### Generate .mind File from Image
```bash
mindar-compiler -i assets/target.jpg -o assets/target.mind
```

#### Expected Output
```
MindAR Compiler v1.x.x
Compiling image: assets/target.jpg
✓ Training data generated successfully
Output: assets/target.mind
```

### Option 2: Online Compiler

1. Visit: https://www.npmjs.com/package/@mnstc/mindar-compiler
2. Upload your target image (JPG/PNG)
3. Download the generated `.mind` file
4. Place in `assets/target.mind`

### Option 3: Using npx (No Installation)

```bash
npx @mnstc/mindar-compiler -i assets/target.jpg -o assets/target.mind
```

## Image Requirements

Your target image should:
- ✓ Have high contrast and texture
- ✓ Be at least 150x150 pixels
- ✓ Use JPG or PNG format
- ✓ Contain distinctive features
- ✓ Avoid pure solid colors
- ✓ Work well in various lighting

### Example Good Target Images
- QR codes
- Product packaging with logos
- Magazine covers
- Textured photographs
- Business cards with unique designs

### Example Bad Target Images
- Solid white/black background
- Blurry images
- Very dark images
- Repetitive patterns only
- Low contrast

## Testing Your Setup

1. Place your target image in `assets/target.jpg`
2. Generate `.mind` file using compiler
3. Open application in mobile browser
4. Hold target image in front of camera
5. Video should play when image detected

## Troubleshooting

### Compiler Not Finding Image
```bash
# Make sure file exists
ls -la assets/target.jpg

# Try absolute path
mindar-compiler -i /full/path/to/target.jpg -o assets/target.mind
```

### Invalid .mind File
- Ensure original image is high quality
- Try different image with more distinct features
- Check image dimensions (should be at least 150x150)

### Detection Not Working in Browser
- Verify `.mind` file was created (check file size > 1KB)
- Ensure target image is similar to original
- Try with different lighting/angles
- Check browser console for errors

## Advanced: Multiple Targets

To track multiple images:

1. Generate `.mind` file for each image
2. Modify `index.html` to include multiple targets
3. Create separate video planes for each target
4. Update JavaScript to handle multiple detections

Example:
```html
<a-entity mindar-image-target="targetIndex: 0">
    <!-- First target content -->
</a-entity>
<a-entity mindar-image-target="targetIndex: 1">
    <!-- Second target content -->
</a-entity>
```

Combine multiple `.mind` files:
```bash
mindar-compiler -i image1.jpg image2.jpg -o combined.mind
```

## File Formats

### MindAR .mind Format
- Binary file format
- Contains training features
- Generated by official compiler
- Not human-readable

### Video Format Support
- MP4 (H.264) - Recommended
- WebM
- Ogg

### Image Format Support
- JPG - Most common
- PNG - With transparency
- GIF - Static only

## Performance Optimization

### Image Optimization
```bash
# Reduce JPG file size
convert target.jpg -quality 85 target-optimized.jpg

# Reduce dimensions if needed
convert target.jpg -resize 500x500 target-resized.jpg
```

### Video Optimization
```bash
# Compress MP4
ffmpeg -i video.mp4 -c:v libx264 -preset slow -crf 23 video-compressed.mp4

# Reduce bitrate
ffmpeg -i video.mp4 -b:v 1000k video-1mbps.mp4
```
